Site estático, Blog, Loja e Proxy reverso

Parte importante de toda automatização é entender como o processo funciona de forma manual. Aqui nós vamos contruir um Site estático, com um Blog (WordPress), uma loja (com Magento) e ainda configurar um proxy reverso.

Estes são os passos:

• Criar um webserver centos7 na AWS • Configurar uma loja com Magento • Configurar um blog com WordPress • Configurar o Tomcat • Criar um site estático • Criar um domínio e ajustar as entradas de DNS • Criar certificado SSL wildcard

CentOS 7, DNS e segurança

yum update
yum -y install epel-release
yum -y install nginx
systemctl start nginx
systemctl enable nginx
yum -y install vim

Então subi uma máquina CentOS 7 na AWS e configurei o SSM, para ficar independente da chave.pem e me conectar de qualquer lugar. É um ponto de segurança importante para não perder o controle de acesso na máquina e dar mais autonomia ao administrador. »» https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/session-manager.html

Em seguida me preocupei com o domínio, então cadastrei um gratuito no freenom (.tk). Ao colocar os dados é importante simular um endereço dos EUA para que consiga obter o domínio. Se você não tem uma casa nos EUA pode simular um endereço facilmente no google. O SG na AWS deve ser configurado conforme a necessidade: 80 para HTTP, 443 para HTTPS, 8080 para Tomcat e 22 caso ainda queira deixar SSH aberto em alternativa ao SSM. Caso queira ainda dar um pouco mais de segurança, recomendo o uso de um Firewall dentro da CentOS, funcionando como um Firewall de borda.

Logo após é preciso fazer o redirecionamento do DNS com o Route 53 na AWS. Este serviço vai garantir a comunicação do endereço hospedado, então crie uma “zona hospedada” com o domínio americano gratuito (rs). Outro detalhe interessante é usar um ElasticIP na sua instância para que as entradas não se percam ao reiniciar a máquina por exemplo.

Site estático

Os códigos das páginas que serão exibidos no navegador pelo NGINX ficam em /var/www/html . Então para o site estático criaremos um novo diretório com “mkdir site” e em seguida vamos apenas escrever “oi” em html com o comando “echo” :

echo "<center><h1>oi</h1><center>" > /var/www/html/index.html

E depois criaremos um arquivo de configuração no NGINX para este site estático em /etc/nginx/conf.d/, criando o arquivo site.conf:

echo "<center><h1>oi</h1><center>" > /var/www/html/index.html

E depois criaremos um arquivo de configuração no NGINX para este site estático em /etc/nginx/conf.d/, criando o arquivo site.conf:

server {
   listen  80;
   server_name site-barbosa.barbosateam.tk;
   error_log /var/log/nginx/site/error.log;
   location / {
       root  /var/www/html/site/;
       index  index.html index.htm;
   }
   error_page  500 502 503 504  /50x.html;
}

Eu particularmente prefiro separar os logs por blocos do que depois caçar agulha em palheiro. Por isso que criei o diretório site para o error.log. Qualquer eventual problema é soltar o comando tail neste cara e debugar.

Logo após, apenas adicionamos o apontamento no Route 53 da AWS, ou seja, criamos um registro apontando o DNS para o IP da CentOS7.

Tomcat

O Tomcat foi instalado para a prática de proxy reverso, que inibe a entrada de possíveis invasores ao fazer requisições diretamente no servidor.

#Necessário para o Tomcat
sudo yum install java-1.7.0-openjdk-devel
#Criando um usuário para o Tomcat
sudo groupadd tomcat
#WGET da massa, necessário para transferência de pacotes
sudo yum install wget
#Download do Tomcat
wget https://www.apache.org/dist/tomcat/tomcat-8/v8.5.49/bin/apache-tomcat-8.5.49.tar.gz /tmp
#Descompactando e mandando para /opt
cd /tmp
tar xf apache-tomcat-8.5.49.tar.gz
mv apache-tomcat-8.5.49 /opt/tomcat
#Alterando a permissão do arquivo
sudo useradd -M -s /bin/nologin -g tomcat -d /opt/tomcat tomcat
#Concedendo ao grupo do Tomcat todo o diretório de instalação
sudo chgrp -R tomcat /opt/tomcat
#Tornando o user tomcat dono dos diretórios
sudo chown -R tomcat webapps/ work/ temp/ logs/
#Criando arquivo de inicialização
sudo vim /etc/systemd/system/tomcat.service

O arquivo deve conter o seguinte conteúdo:

# Systemd unit file for tomcat
[Unit]
Description=Apache Tomcat Web Application Container
After=syslog.target network.target

[Service]
Type=forking

Environment=JAVA_HOME=/usr/lib/jvm/jre
Environment=CATALINA_PID=/opt/tomcat/temp/tomcat.pid
Environment=CATALINA_HOME=/opt/tomcat
Environment=CATALINA_BASE=/opt/tomcat
Environment='CATALINA_OPTS=-Xms512M -Xmx1024M -server -XX:+UseParallelGC'
Environment='JAVA_OPTS=-Djava.awt.headless=true -Djava.security.egd=file:/dev/./urandom'

ExecStart=/opt/tomcat/bin/startup.sh
ExecStop=/bin/kill -15 $MAINPID

User=tomcat
Group=tomcat
UMask=0007
RestartSec=10
Restart=always

[Install]
WantedBy=multi-user.target

Depois de salvar o arquivo:

#Reload no systemd
sudo systemctl daemon-reload
#Start no tomcat
sudo systemctl start tomcat
#Ativando serviço ao iniciar o sistema
sudo systemctl enable tomcat

Então lá no diretório dos arquivos de configuração, criamos o proxy reverso:

server {
 listen 80;
 server_name tomcat-barbosa.barbosateam.tk;
 access_log /var/log/nginx/tomcat/access.log;
 error_log /var/log/nginx/tomcat/error.log;


 location / {
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header Host $host;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

  proxy_pass http://127.0.0.1:8080;
 }
}

Com esta configuração será possível acessar o Tomcat na porta 80.

WordPress

A primeira coisa é criar um banco de dados com o WordPress:

#Criando o Banco
CREATE DATABASE wordpress;
#Criando um usuario para o banco com o nome wordpressuser e a senha senhatemporaria.
CREATE USER wordpressuser@localhost IDENTIFIED BY 'senhatemporaria';
#Privilegio para o usuário que criamos anteriomente
GRANT ALL PRIVILEGES ON wordpress.* TO wordpressuser@localhost IDENTIFIED BY 'senhatemporaria';
#Modificando
FLUSH PRIVILEGES;
#Saindo do MySQL
exit

Não se esqueça que cada macaco deve estar no seu galho, então crie um diretório chamado do que você quiser lá em /var/www/html, no caso chamei o meu de Blog.

Aqui nesta parte eu acabei instalando o PHP sem especificar que deveria ser o PHP7. Isto porque a útima versão do WordPress exige o PHP7. Para não cometer este erro e ser obrigado a fazer o upgrade do PHP instale a versão correta com este comando:

yum -y install php70w-fpm php70w-mcrypt php70w-curl php70w-cli php70w-mysql php70w-gd php70w-xsl php70w-json php70w-intl php70w-pear php70w-devel php70w-mbstring php70w-zip php70w-soap

Mas se por um deslize:

Instalou o PHP5, faça o upgrade da seguinte forma:

#Remova o PHP5
 yum remove php-common php-fpm php-cli -y
#Instale o PHP7
yum -y install php70w-fpm php70w-mcrypt php70w-curl php70w-cli php70w-mysql php70w-gd php70w-xsl php70w-json php70w-intl php70w-pear php70w-devel php70w-mbstring php70w-zip php70w-soap
#Reinicie o NGINX
systemctl restart nginx
#Reinicie o PHP
systemctl restart php-fpm

E então instale o WordPress:

#Indo para a ṕasta de temporários para baixar o pacote
cd /tmp
#Fazendo o download
wget http://wordpress.org/latest.tar.gz
#Descompactando o pacote
tar xf latest.tar.gz
#Movendo a pasta para o diretório ao qual o NGINX vai ler
mv wordpress /var/www/html/blog
#Vamos alterar a permissão do diretório do blog
sudo chown -R nginx:nginx /var/www/html/blog/*
#Vamos ir até o diretorio do blog e fazer uma copia do arquivo de configuração chamado wp-config.php.
cp wp-config-sample.php wp-config.php

Agora vamos alterar o arquivo wp-config.php com os dados do Banco:

// ** MySQL settings - You can get this info from your web host ** //
/** The name of the database for WordPress */
define('DB_NAME', 'wordpress');

/** MySQL database username */
define('DB_USER', 'wordpressuser');

/** MySQL database password */
define('DB_PASSWORD', 'password');

E então podemos agora criar o arquivo de configuração em /etc/nginx/conf.d/blog.conf:

server {
server_name blog-barbosa.barbosateam.tk;
   root /var/www/html/blog/;
   index index.html index.htm index.php;
   access_log /var/log/nginx/blog/access.log;
   error_log /var/log/nginx/blog/error.log;
 location = /favicon.ico {
       log_not_found off;
       access_log off;
   }
   location = /robots.txt {
       allow all;
       log_not_found off;
       access_log off;
   }
   location / {
       try_files $uri $uri/ /index.php?$args;
   }
   location ~ \.php$ {
       try_files $uri =404;
       fastcgi_pass unix:/run/php/php-fpm.sock;
       fastcgi_index   index.php;
       fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
       include fastcgi_params;
   }
   location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
       expires max;
       log_not_found off;
   }
}

E conceder permissão no sistema:

#Ajustes no Firewall SElinux
sudo setsebool -P httpd_can_network_connect on
getenforce
chcon -Rt httpd_sys_content_t /var/www/html/blog
#Verificando se está tudo OK com o NGINX
sudo nginx -t
#Restart no serviço NGINX
sudo service nginx restart

Magento

É preciso configurar o arquivo php.ini:

vim /etc/php.ini

#Descomente as linhas
cgi.fix_pathinfo=0
session.save_path = "/var/lib/php/session"
#Altere os valores
memory_limit = 512M
max_execution_time = 1800
zlib.output_compression = On

Em /etc/php-fpm.d/www.conf :

#É necessário setar o socket para o php-fpm
listen = /var/run/php/php-fpm.sock

Adicionando o grupo, usuário, nginx e a permissão.

listen.owner = nginx
listen.group = nginx
listen.mode = 0660
#Descomente as seguintes linhas
env[HOSTNAME] = $HOSTNAME
env[PATH] = /usr/local/bin:/usr/bin:/bin
env[TMP] = /tmp
env[TMPDIR] = /tmp
env[TEMP] = /tmp
#Crie dois diretórios
mkdir -p /var/lib/php/session/
chown -R nginx:nginx /var/lib/php/session/
#Criando novo diretório para o php-fpm
mkdir -p /run/php/
chown -R nginx:nginx /run/php/
#Iniciando o php-fpm
systemctl start php-fpm
systemctl enable php-fpm

Como troubleshooting, verifique se há algum erro e aonde está o php-fpm.sock:

netstat -pl | grep php-fpm.sock

E então crie o Banco para a Loja com Magento:

create database magentodb;
create user magentouser@localhost identified by 'Magento123@';
grant all privileges on magentodb.* to magentouser@localhost identified by 'Magento123@';
flush privileges;

No diretório do Magento instalamos o Composer:

curl -sS https://getcomposer.org/installer | sudo php -- --install-dir=/usr/bin --filename=composer
#E podemos fazer o download dos plugins
composer -v

Finalmente o download do Magento:

cd /var/www/
mkdir html
cd html
wget https://github.com/magento/magento2/archive/2.1.zip
#Descompactando e renomeando
unzip 2.1.zip
mv magento2-2.1 magento2
#Realizando a instalação das dependências
cd magento2
composer install -v

No diretório de do Magento, setamos a configuração do Magento. O seguinte bloco pode ser adicionado a linha de comando, com as respectivas informações do seu ambiente, é claro.

bin/magento setup:install --backend-frontname="adminlogin" \
--key="biY8vdWx4w8KV5Q59380Fejy36l6ssUb" \
--db-host="localhost" \
--db-name="magentodb" \
--db-user="magentouser" \
--db-password="Magento123@" \
--language="en_US" \
--currency="USD" \
--timezone="America/Sao_Paulo" \
--use-rewrites=1 \
--use-secure=0 \
--base-url="http://loja-barbosa.barbosateam.tk" \
--base-url-secure="http://loja-barbosa.barbosateam.tk" \
--admin-user=adminuser \
--admin-password=admin123@ \
--admin-email=admin@newmagento.com \
--admin-firstname=admin \
--admin-lastname=user \
--cleanup-database

Damos a permissão para a aplicação e para o NGINX:

chmod 700 /var/www/magento2/app/etc
chown -R nginx:nginx /var/www/magento2

E FINALMENTE configuramos o magento.conf:

upstream fastcgi_backend {
        server  unix:/run/php/php-fpm.sock;
}

server {

        listen 80;
        server_name loja-barbosa.barbosateam.tk;
        set $MAGE_ROOT /var/www/html/magento2;
        set $MAGE_MODE developer;
        include /var/www/html/magento2/nginx.conf.sample;
}
#Verificando se está tudo OK
nginx -t
#Reiniciando o NGINX
systemctl restart nginx
